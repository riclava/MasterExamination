# 计算机网络满分知识点与练习题汇总

## 1. 网络封装与传输效率

### 必须掌握

数据从应用层向下传输时，每一层都会添加协议开销。

常考公式：

```text
传输效率 = 应用层有效数据 / 实际传输总数据 × 100%
```

实际传输总数据包括：

```text
应用层数据 + TCP头部 + IP头部 + 数据链路层头部 + 数据链路层尾部
```

### 练习题

**题 1：**
应用层发送 1000 字节数据，TCP 头部 20 字节，IP 头部 20 字节，数据链路层头部 14 字节，尾部 4 字节，求传输效率。

答案：

```text
总长度 = 1000 + 20 + 20 + 14 + 4 = 1058 字节
传输效率 = 1000 / 1058 ≈ 94.5%
```

**题 2：**
应用层数据为 800 字节，各层协议总开销为 80 字节，求传输效率。

答案：

```text
传输效率 = 800 / (800 + 80) = 800 / 880 ≈ 90.9%
```

**题 3：**
如果应用层数据越小，协议头部不变，传输效率会怎样变化？

答案：

```text
应用层数据越小，协议开销占比越大，传输效率越低。
```

## 2. CIDR、网络前缀与子网掩码

### 必须掌握

CIDR 形式：

```text
x.x.x.x/n
```

其中：

```text
n 表示网络前缀长度
主机位 = 32 - n
```

例如：

```text
/21 表示前 21 位是网络位，后 11 位是主机位
```

### 练习题

**题 1：**
192.168.0.0/24 中，网络前缀长度是多少？主机位有多少位？

答案：

```text
网络前缀长度 = 24
主机位 = 32 - 24 = 8 位
```

**题 2：**
10.0.0.0/20 的主机位有多少位？

答案：

```text
主机位 = 32 - 20 = 12 位
```

**题 3：**
/26 对应的子网掩码是多少？

答案：

```text
/26 = 11111111.11111111.11111111.11000000
子网掩码 = 255.255.255.192
```

## 3. 子网划分与网络号计算

### 必须掌握

根据子网掩码判断网络号。

常用方法：

```text
块大小 = 256 - 子网掩码最后一个非 255 字节
```

例如：

```text
255.255.255.224
块大小 = 256 - 224 = 32
```

所以子网范围为：

```text
0-31
32-63
64-95
96-127
...
```

### 练习题

**题 1：**
IP 地址 192.168.1.45，子网掩码 255.255.255.224，求网络号。

答案：

```text
块大小 = 256 - 224 = 32
45 落在 32-63
网络号 = 192.168.1.32
```

**题 2：**
IP 地址 192.168.2.130，子网掩码 255.255.255.192，求网络号。

答案：

```text
块大小 = 256 - 192 = 64
130 落在 128-191
网络号 = 192.168.2.128
```

**题 3：**
IP 地址 10.1.5.200，子网掩码 255.255.255.128，求网络号。

答案：

```text
块大小 = 256 - 128 = 128
200 落在 128-255
网络号 = 10.1.5.128
```

## 4. 主机数量计算

### 必须掌握

主机位为 n 时：

```text
可用主机数 = 2^n - 2
```

减去 2 是因为：

```text
一个网络地址
一个广播地址
```

### 练习题

**题 1：**
/24 网络最多有多少个可用主机地址？

答案：

```text
主机位 = 32 - 24 = 8
可用主机数 = 2^8 - 2 = 254
```

**题 2：**
/27 网络最多有多少个可用主机地址？

答案：

```text
主机位 = 32 - 27 = 5
可用主机数 = 2^5 - 2 = 30
```

**题 3：**
/21 网络最多有多少个可用主机地址？

答案：

```text
主机位 = 32 - 21 = 11
可用主机数 = 2^11 - 2 = 2046
```

## 5. 路由器转发与路由表查询

### 必须掌握

如果一个 IP 数据报经过 n 个路由器：

```text
网络设备总数 = 源主机 + n 个路由器 + 目的主机
路由表查询次数 = n 次
```

因为每个路由器转发时都要查询一次路由表。

### 练习题

**题 1：**
一个数据报从源主机到目的主机，中间经过 3 个路由器，共经过多少个网络设备？路由表查询几次？

答案：

```text
网络设备数 = 1 + 3 + 1 = 5
路由表查询次数 = 3 次
```

**题 2：**
一个数据报经过 5 个路由器，沿途共有多少个设备？

答案：

```text
1 个源主机 + 5 个路由器 + 1 个目的主机 = 7 个设备
```

**题 3：**
主机发送 IP 数据报时，目的主机是否查询路由表来转发该数据报？

答案：

```text
不查询。目的主机是接收者，不再转发该数据报。
```

## 6. 交换机、冲突域与广播域

### 必须掌握

交换机特点：

```text
每个交换机端口是一个独立冲突域
默认未划分 VLAN 时，整个交换机属于一个广播域
```

例如 48 端口交换机：

```text
每个冲突域包含 1 个端口
广播域包含 48 个端口
```

### 练习题

**题 1：**
一台 24 端口交换机，未划分 VLAN，每个冲突域包含几个端口？广播域包含几个端口？

答案：

```text
每个冲突域包含 1 个端口
广播域包含 24 个端口
```

**题 2：**
一台 48 端口交换机划分为 4 个 VLAN，每个 VLAN 12 个端口，共有几个广播域？

答案：

```text
4 个广播域
```

**题 3：**
交换机能隔离冲突域还是广播域？

答案：

```text
默认能隔离冲突域，不能隔离广播域。
划分 VLAN 后才能隔离广播域。
```

## 7. MAC 地址表自学习

### 必须掌握

交换机自学习依据：

```text
源 MAC 地址
```

转发时查询：

```text
目的 MAC 地址
```

完整过程：

```text
收到数据帧
读取源 MAC 地址
记录 源 MAC 地址 → 入端口
再根据目的 MAC 地址决定转发端口
```

### 练习题

**题 1：**
交换机学习 MAC 地址表时依据的是源 MAC 还是目的 MAC？

答案：

```text
源 MAC 地址。
```

**题 2：**
交换机根据什么字段决定数据帧从哪个端口转发出去？

答案：

```text
目的 MAC 地址。
```

**题 3：**
如果交换机不知道目的 MAC 地址对应哪个端口，会怎么处理？

答案：

```text
会泛洪，即向除入端口外的其他端口转发。
```

## 8. MTU 最大传输单元

### 必须掌握

MTU：

```text
Maximum Transmission Unit
最大传输单元
```

含义：

```text
数据链路层能够承载的最大数据部分长度
```

在 IP 分片题中，通常理解为：

```text
一个 IP 数据报在该链路上允许的最大总长度
```

以太网常见 MTU：

```text
1500 字节
```

### 练习题

**题 1：**
以太网常见 MTU 是多少？

答案：

```text
1500 字节
```

**题 2：**
如果 IP 数据报长度大于下一跳网络的 MTU，会发生什么？

答案：

```text
需要进行 IP 分片。
```

**题 3：**
MTU 为 1500 字节，IP 头部 20 字节，每个分片最多能装多少字节数据？

答案：

```text
1500 - 20 = 1480 字节
```

## 9. IP 分片计算

### 必须掌握

IP 分片三条规则：

```text
1. 每个分片都有 IP 头部
2. 每个分片总长度不能超过 MTU
3. 除最后一个分片外，数据部分长度必须是 8 字节的整数倍
```

核心公式：

```text
每片最大数据长度 = MTU - IP头部长度
```

如果不是 8 的倍数，需要向下取最接近的 8 的倍数。

### 练习题

**题 1：**
一个 IP 数据报头部 20 字节，数据部分 3000 字节，经过 MTU 为 1500 的网络，需要分成几片？每片总长度是多少？

答案：

```text
每片最大数据 = 1500 - 20 = 1480 字节

3000 = 1480 + 1480 + 40

所以分 3 片：
第 1 片总长度 = 20 + 1480 = 1500
第 2 片总长度 = 20 + 1480 = 1500
第 3 片总长度 = 20 + 40 = 60
```

**题 2：**
MTU 为 576 字节，IP 头部 20 字节，每片最大数据长度应该取多少？

答案：

```text
576 - 20 = 556
556 不是 8 的倍数
向下取 552
所以每片最大数据长度 = 552 字节
```

**题 3：**
一个数据部分为 2000 字节的 IP 数据报，IP 头部 20 字节，经过 MTU 为 576 的网络，最终分几片？

答案：

```text
每片最大数据 = 552 字节

2000 = 552 + 552 + 552 + 344

所以分 4 片。
```

## 10. BGP 协议

### 必须掌握

BGP 是：

```text
Border Gateway Protocol
边界网关协议
```

核心特点：

```text
BGP 是自治系统 AS 之间的路由协议
BGP 使用 TCP 端口 179
BGP 采用路径向量算法
BGP 不是广播全部路由，而是通告最佳路由
```

### 练习题

**题 1：**
BGP 是自治系统内部协议还是自治系统之间协议？

答案：

```text
自治系统之间的路由协议。
```

**题 2：**
BGP 使用 TCP 还是 UDP？

答案：

```text
TCP，端口号 179。
```

**题 3：**
BGP 使用什么路由算法？

答案：

```text
路径向量算法。
```

## 11. TCP 可靠传输机制

### 必须掌握

TCP 可靠传输依靠：

```text
序列号
确认号
校验和
超时重传
滑动窗口
流量控制
拥塞控制
```

不属于 TCP 可靠传输机制的是：

```text
广播重传
```

因为 TCP 是点对点、面向连接的协议。

### 练习题

**题 1：**
TCP 如何判断数据是否按序到达？

答案：

```text
通过序列号。
```

**题 2：**
TCP 发送方长时间没有收到确认，会怎么做？

答案：

```text
超时重传。
```

**题 3：**
广播重传是不是 TCP 可靠传输机制？

答案：

```text
不是。TCP 是面向连接的点对点协议，不支持广播重传。
```

## 12. TCP 序列号回绕与最大速率

### 必须掌握

为了避免序列号回绕导致混淆，在 MSL 时间内不能重复使用序列号。

核心公式：

```text
最大速率 = 2^序列号位数 × 每段数据字节数 × 8 / MSL
```

### 练习题

**题 1：**
序列号字段长度为 4 位，每段数据 1000 字节，MSL 为 100 秒，最大传输速率是多少？

答案：

```text
序列号个数 = 2^4 = 16
总数据量 = 16 × 1000 × 8 = 128000 bit
最大速率 = 128000 / 100 = 1280 bps
```

**题 2：**
序列号字段长度为 5 位，每段数据 500 字节，MSL 为 80 秒，最大传输速率是多少？

答案：

```text
序列号个数 = 2^5 = 32
总数据量 = 32 × 500 × 8 = 128000 bit
最大速率 = 128000 / 80 = 1600 bps
```

**题 3：**
为什么序列号字段太短会限制最大传输速率？

答案：

```text
因为序列号数量有限，发送过快会在旧数据还未消失前重复使用序列号，导致接收方混淆新旧数据。
```

## 13. TCP 拥塞控制：慢启动与拥塞避免

### 必须掌握

TCP 超时触发拥塞时：

```text
ssthresh = 当前 cwnd / 2
cwnd = 1 MSS
```

之后：

```text
cwnd < ssthresh：慢启动，指数增长
cwnd >= ssthresh：拥塞避免，线性增长
```

### 练习题

**题 1：**
某 TCP 连接 cwnd = 20KB 时发生超时拥塞，求新的 ssthresh 和 cwnd。

答案：

```text
ssthresh = 20KB / 2 = 10KB
cwnd = 1KB 或 1 MSS
```

**题 2：**
拥塞后 cwnd = 1KB，ssthresh = 8KB，接下来 3 个 RTT 后 cwnd 分别是多少？

答案：

```text
第 1 个 RTT 后：2KB
第 2 个 RTT 后：4KB
第 3 个 RTT 后：8KB
```

**题 3：**
cwnd 达到 ssthresh 后，TCP 进入什么阶段？

答案：

```text
进入拥塞避免阶段，窗口线性增长。
```

## 14. SAP 服务访问点

### 必须掌握

SAP：

```text
Service Access Point
服务访问点
```

含义：

```text
相邻两层之间的逻辑接口
上层通过 SAP 使用下层提供的服务
```

常见理解：

```text
端口号可以看作应用层访问传输层服务的标识
```

### 练习题

**题 1：**
SAP 的中文名称是什么？

答案：

```text
服务访问点。
```

**题 2：**
SAP 的作用是什么？

答案：

```text
作为相邻两层之间的逻辑接口，使上层能够访问下层提供的服务。
```

**题 3：**
传输层中常用什么来区分不同应用进程？

答案：

```text
端口号。
```
