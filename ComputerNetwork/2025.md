# 计算机网络满分知识点 + 练习题

## 1. CSMA/CA 与无线局域网

### 必须掌握

CSMA/CA 的中文全称是：

```text
载波监听多路访问 / 冲突避免
```

CA 是：

```text
Collision Avoidance，冲突避免
```

它常用于无线局域网，例如 Wi-Fi。
它和 CSMA/CD 不同：

```text
CSMA/CD：冲突检测，常用于传统有线以太网
CSMA/CA：冲突避免，常用于无线局域网
```

ACK 的作用是确认数据帧已经被正确接收。

### 练习题

1. 填空：CSMA/CA 的中文全称是 ______，其中 CA 表示 ______。

   答案：载波监听多路访问/冲突避免；冲突避免。

2. 选择：CSMA/CA 主要用于哪类网络？

   A. 光纤骨干网
   B. 无线局域网
   C. 电话交换网
   D. 卫星广播网

   答案：B。

3. 简答：为什么无线局域网更适合使用 CSMA/CA，而不是 CSMA/CD？

   答案要点：无线环境中设备难以边发送边检测冲突，且存在隐藏站问题，因此采用冲突避免机制。

## 2. 停止-等待协议与信道利用率

### 必须掌握

停止-等待协议的特点：

```text
发送方每发送一帧，就必须等待接收方 ACK 后才能发送下一帧。
```

所以如果 RTT 很大，信道利用率会很低。

核心公式：

```text
发送时延 = 帧长 / 传输速率

传播时延 = 链路长度 / 信号传播速度

RTT = 2 × 传播时延

停止-等待利用率 = 发送时延 / (发送时延 + RTT)
```

提高利用率的方法：

```text
使用流水线技术
```

例如滑动窗口协议、连续 ARQ 等。

### 练习题

1. 某停止-等待协议中，发送时延为 2ms，RTT 为 18ms，求信道利用率。

   答案：

   ```text
   U = 2 / (2 + 18) = 10%
   ```

2. 某链路长 200km，信号传播速度为 2×10⁵ km/s，求单程传播时延和 RTT。

   答案：

   ```text
   单程传播时延 = 200 / (2×10⁵) = 0.001s = 1ms
   RTT = 2ms
   ```

3. 简答：停止-等待协议为什么效率低？如何提高？

   答案要点：因为每发一帧都要等待 ACK，等待期间信道空闲；可用流水线技术连续发送多个帧。

## 3. 发送时延、传播时延、RTT

### 必须掌握

这是计算题高频点。

```text
发送时延：把数据全部推送到链路上的时间
传播时延：信号从发送端传播到接收端的时间
RTT：往返传播时间
```

不要混淆：

```text
发送时延和数据大小、传输速率有关
传播时延和物理距离、信号传播速度有关
```

### 练习题

1. 帧长为 1000 字节，传输速率为 8Mbps，求发送时延。

   答案：

   ```text
   1000 字节 = 8000 bit
   发送时延 = 8000 / (8×10⁶) = 0.001s = 1ms
   ```

2. 链路长度为 100km，信号传播速度为 2×10⁵ km/s，求传播时延。

   答案：

   ```text
   传播时延 = 100 / (2×10⁵) = 0.0005s = 0.5ms
   ```

3. 如果传播时延等于发送时延，帧长为 2000 字节，传播时延为 0.5ms，求传输速率。

   答案：

   ```text
   2000 字节 = 16000 bit
   R = 16000 / 0.0005 = 32Mbps
   ```

## 4. DNS 递归解析

### 必须掌握

递归解析过程：

```text
客户端向本地域名服务器发送查询请求；
本地域名服务器负责继续向其他 DNS 服务器查询；
最终把结果返回给客户端。
```

核心关键词：

```text
客户端
本地域名服务器
递归查询
最终返回客户端
```

递归查询和迭代查询区别：

```text
递归查询：客户端只问一次，服务器帮它查到底。
迭代查询：服务器告诉客户端下一步该问谁。
```

### 练习题

1. 填空：DNS 递归解析中，客户端通常首先向 ______ 服务器发送查询请求。

   答案：本地域名服务器。

2. 填空：DNS 递归解析中，最终查询结果返回给 ______。

   答案：客户端。

3. 简答：递归查询和迭代查询有什么区别？

   答案要点：递归查询由服务器代替客户端完成完整查询；迭代查询中服务器只返回下一步查询线索。

## 5. TCP 序号与确认号

### 必须掌握

TCP 中：

```text
SEQ：当前发送数据的第一个字节序号
ACK：期望收到的下一个字节序号
```

计算公式：

```text
返回 ACK = 对方发送的 SEQ + 有效载荷长度
```

例如：

```text
A 发送 seq = 300，有效载荷 100 字节
B 返回 ack = 300 + 100 = 400
```

### 练习题

1. A 向 B 发送 TCP 报文段，seq=500，有效载荷为 200 字节。B 返回的 ack 应为多少？

   答案：

   ```text
   ack = 500 + 200 = 700
   ```

2. A 发送 seq=1000，ack=300，数据长度为 500 字节。B 收到后确认，B 的 ack 是多少？

   答案：

   ```text
   ack = 1000 + 500 = 1500
   ```

3. 判断：TCP 的 ACK 号表示“已经收到的最后一个字节序号”。

   答案：错误。ACK 表示期望收到的下一个字节序号。

## 6. UDP 端口号作用

### 必须掌握

UDP 端口号用于：

```text
区分同一主机上的不同应用进程
```

几个常见概念要分清：

```text
IP 地址：标识主机
端口号：标识应用进程
MAC 地址：标识局域网中的网卡
```

UDP 的特点：

```text
无连接
不保证可靠传输
开销小
速度快
常用于 DNS、视频、语音、实时通信等
```

### 练习题

1. 选择：UDP 端口号的主要作用是？

   A. 标识主机
   B. 区分不同应用进程
   C. 实现拥塞控制
   D. 实现路由选择

   答案：B。

2. 判断：UDP 通过端口号来区分不同主机。

   答案：错误。主机由 IP 地址区分，应用进程由端口号区分。

3. 简答：为什么 DNS 常使用 UDP？

   答案要点：DNS 查询数据量小，要求响应快，UDP 开销低，适合简单请求-响应场景。

## 7. CIDR 与子网范围计算

### 必须掌握

CIDR 表示法：

```text
IP地址/网络前缀长度
```

例如：

```text
192.168.1.0/22
```

/22 表示前 22 位是网络号。

常见计算方法：

```text
子网掩码第三字节 = 252
块大小 = 256 - 252 = 4
```

所以：

```text
192.168.0.0/22 覆盖范围：
192.168.0.0 ~ 192.168.3.255
```

做题时要会判断两个 IP 是否属于同一个网段。

### 练习题

1. 判断：192.168.2.1 和 192.168.3.200 是否属于 192.168.0.0/22？

   答案：属于。范围是 192.168.0.0 到 192.168.3.255。

2. 选择：下列哪个地址属于 192.168.4.0/22？

   A. 192.168.3.255
   B. 192.168.4.1
   C. 192.168.8.1
   D. 192.168.2.2

   答案：B。192.168.4.0/22 范围是 192.168.4.0 到 192.168.7.255。

3. 计算：172.16.8.0/21 的地址范围是多少？

   答案：

   ```text
   /21 对应第三字节掩码 248
   块大小 = 256 - 248 = 8
   范围：172.16.8.0 ~ 172.16.15.255
   ```

## 8. OSPF 路由协议

### 必须掌握

OSPF 是：

```text
内部网关协议 IGP
链路状态路由协议
使用 Dijkstra 算法计算最短路径
适合较大型网络
```

对比记忆：

```text
RIP：内部网关协议，距离向量算法
OSPF：内部网关协议，链路状态算法
BGP：外部网关协议
```

### 练习题

1. 选择：OSPF 属于哪类协议？

   A. 外部网关协议
   B. 内部网关协议
   C. 应用层协议
   D. 传输层协议

   答案：B。

2. 选择：OSPF 使用什么算法计算最短路径？

   A. Bellman-Ford
   B. Dijkstra
   C. CRC
   D. RSA

   答案：B。

3. 判断：OSPF 是基于距离向量的路由协议。

   答案：错误。OSPF 是链路状态协议。

源端口
目的端口

## 10. STP 生成树协议

### 必须掌握

STP 的作用：

```text
防止二层交换网络出现环路
```

核心过程：

```text
选举根桥
选择根端口
选择指定端口
阻塞部分冗余链路
形成无环树状拓扑
```

标准答题模板：

```text
生成树协议 STP 是以太网中用于防止链路层环路的协议。
它通过选举根桥、根端口和指定端口，逻辑上阻塞部分冗余链路，
使网络形成无环的树状拓扑。当主链路故障时，可以启用备用链路恢复通信。
```

### 练习题

1. 名词解释：什么是 STP？

   答案要点：用于防止二层环路，通过阻塞冗余链路形成无环拓扑。

2. 填空：STP 通过选举 ******、****** 和 ______ 来形成无环拓扑。

   答案：根桥、根端口、指定端口。

3. 判断：STP 会物理断开冗余链路。

   答案：错误。STP 是逻辑上阻塞冗余链路。

## 11. 流量控制

### 必须掌握

流量控制的目的：

```text
防止发送方发送太快，导致接收方缓冲区溢出。
```

TCP 中常用：

```text
滑动窗口机制
```

注意区分：

```text
流量控制：控制发送方，保护接收方
拥塞控制：控制发送方，保护网络
```

标准答题模板：

```text
流量控制是调节发送方发送速率的机制，
目的是防止接收方缓冲区溢出。
在 TCP 中，通常通过滑动窗口实现，
接收方通过窗口大小告知发送方还能接收多少数据。
```

### 练习题

1. 名词解释：什么是流量控制？

   答案要点：调节发送速率，防止接收方缓冲区溢出。

2. 选择：TCP 中实现流量控制的主要机制是？

   A. 路由选择
   B. 滑动窗口
   C. DNS 查询
   D. ARP 广播

   答案：B。

3. 简答：流量控制和拥塞控制有什么区别？

   答案要点：流量控制保护接收方，防止接收方处理不过来；拥塞控制保护网络，防止网络过载。

## 12. TCP 拥塞控制

### 必须掌握

TCP 拥塞控制四个核心机制：

```text
慢开始
拥塞避免
快重传
快恢复
```

窗口变化：

```text
慢开始：指数增长
1 → 2 → 4 → 8 → 16

拥塞避免：线性增长
8 → 9 → 10 → 11 → 12
```

收到 3 个重复 ACK：

```text
触发快重传
ssthresh = 当前拥塞窗口的一半
通常进入快恢复/拥塞避免
```

超时重传和 3 个重复 ACK 要区分：

```text
超时：通常认为拥塞严重，窗口回到较小值
3 个重复 ACK：通常认为只是个别报文丢失，执行快重传和快恢复
```

### 练习题

1. TCP 拥塞窗口从 1 开始，慢开始增长，经过 4 个 RTT 后窗口大小是多少？

   答案：

   ```text
   1 → 2 → 4 → 8 → 16
   第 4 个 RTT 后为 16
   ```

2. 当前拥塞窗口 cwnd = 20 MSS，收到 3 个重复 ACK 后，ssthresh 应设置为多少？

   答案：

   ```text
   ssthresh = 20 / 2 = 10 MSS
   ```

3. 某 TCP 连接在 cwnd=16K 时收到 3 个重复 ACK，随后进入拥塞避免。若窗口从 8K 开始线性增长，5 个 RTT 后窗口是多少？

   答案：

   ```text
   8K → 9K → 10K → 11K → 12K → 13K
   第 5 个 RTT 后为 13K
   ```

## 13. 序号空间、MSL 与最大数据率

### 必须掌握

这类题的核心逻辑：

```text
序号位数决定最多能区分多少个报文段。
```

如果序号为 n 位：

```text
可用序号数量 = 2^n
```

最大数据率公式：

```text
最大数据率 = 2^n × 最大报文段长度 × 8 / MSL
```

单位注意：

```text
字节要转换成 bit
1 字节 = 8 bit
```

### 练习题

1. 某协议序号为 4 位，最大报文段长度 512 字节，MSL 为 60s，求最大数据率。

   答案：

   ```text
   2^4 = 16
   512 字节 = 4096 bit
   最大数据率 = 16 × 4096 / 60 ≈ 1092.27 bps
   ```

2. 某协议序号为 8 位，最大报文段长度 1000 字节，MSL 为 120s，求最大数据率。

   答案：

   ```text
   2^8 = 256
   1000 字节 = 8000 bit
   最大数据率 = 256 × 8000 / 120 ≈ 17066.67 bps
   ```

3. 判断：序号位数越多，可支持的最大数据率通常越高。

   答案：正确。序号空间越大，在 MSL 内可区分的数据量越多。
