【计算机网络】【总览】计算机网络第一优先级是发送时延、传播时延、RTT、停止等待、滑动窗口、CIDR、IP 分片、TCP 序号、拥塞控制、CSMA/CD、距离向量路由计算。
【计算机网络】【总览】计算机网络第二优先级是 TCP/UDP 区别、IP/MAC/端口/Socket、交换机/路由器/集线器、VLAN、ARP、STP、RIP/OSPF/BGP、DNS、NAT、防火墙、数字签名。
【计算机网络】【总览】计算机网络名词解释高频点包括协议三要素、CIDR、MTU、NAT、VLAN、STP、OSPF、BGP、TCP 流量控制、TCP 拥塞控制、数字签名、防火墙。
【计算机网络】【公式】发送时延公式：$\text{发送时延}=\frac{\text{数据长度(bit)}}{\text{传输速率(bit/s)}}$。
【计算机网络】【公式】数据长度公式：$\text{数据长度}=\text{传输速率}\times\text{发送时延}$。
【计算机网络】【公式】传播时延公式：$\text{传播时延}=\frac{\text{链路长度}}{\text{信号传播速度}}$。
【计算机网络】【公式】RTT 公式：$RTT=2\times\text{单向传播时延}$。
【计算机网络】【公式】停止等待利用率公式：$U=\frac{\text{发送时延}}{\text{发送时延}+RTT}$。
【计算机网络】【公式】滑动窗口充分利用链路条件：$W\times T_t\ge RTT+T_t$。
【计算机网络】【公式】滑动窗口最小窗口公式：$W\ge\frac{RTT+T_t}{T_t}$。
【计算机网络】【公式】传输效率公式：$\text{传输效率}=\frac{\text{有效数据}}{\text{实际传输总数据}}\times100\%$。
【计算机网络】【公式】可用主机数公式：$2^{\text{主机位数}}-2$。
【计算机网络】【公式】子网块大小公式：$256-\text{掩码中最后一个非 255 字节}$。
【计算机网络】【公式】IP 分片最大数据公式：$\text{MTU}-\text{IP 首部长度}$，非最后一片还要向下取 8 的倍数。
【计算机网络】【公式】片偏移公式：$\text{片偏移}=\frac{\text{本片数据起始位置}}8$。
【计算机网络】【公式】GBN 最大发送窗口公式：$W_{\max}=2^n-1$。
【计算机网络】【公式】SR 最大窗口公式：$W_{\max}=2^{n-1}$。
【计算机网络】【公式】TCP 确认号公式：$ACK=\text{对方 SEQ}+\text{有效载荷长度}$。
【计算机网络】【公式】慢启动前 $n$ 轮累计发送公式：$1+2+\cdots+2^{n-1}=2^n-1$。
【计算机网络】【公式】最大数据率公式：$\frac{2^{\text{序号位数}}\times\text{报文段长度(Byte)}\times8}{MSL}$。
【计算机网络】【公式】CSMA/CD 冲突检测条件：$2\times\text{传播时延}\le\text{最小帧发送时延}$。
【计算机网络】【公式】二进制指数退避中第 $n$ 次碰撞取 $k=\min(n,10)$。
【计算机网络】【公式】二进制指数退避随机范围是 $0\sim 2^k-1$ 个时间片。
【计算机网络】【单位】$1\text{ Byte}=8\text{ bit}$。
【计算机网络】【单位】$1\text{ KB}\approx10^3\text{ Byte}$。
【计算机网络】【单位】$1\text{ MB}\approx10^6\text{ Byte}$。
【计算机网络】【单位】$1\text{ Mbps}=10^6\text{ bit/s}$。
【计算机网络】【单位】$1\text{ Gbps}=10^9\text{ bit/s}$。
【计算机网络】【单位】$1\text{ ms}=10^{-3}\text{ s}$。
【计算机网络】【单位】$1\text{ us}=10^{-6}\text{ s}$。
【计算机网络】【协议】网络协议三要素是语法、语义、同步/时序。
【计算机网络】【协议】语法规定数据和控制信息的格式。
【计算机网络】【协议】语义规定控制信息的含义、动作和响应。
【计算机网络】【协议】同步或时序规定事件发生的顺序和时间关系。
【计算机网络】【分层】OSI 分层中下层向上层提供服务。
【计算机网络】【分层】上层通过服务访问点 SAP 使用下层服务。
【计算机网络】【分层】协议是同层实体之间通信的规则。
【计算机网络】【分层】服务是相邻层之间的接口。
【计算机网络】【设备】集线器工作在物理层，转发依据是比特信号。
【计算机网络】【设备】交换机工作在数据链路层，转发依据是 MAC 地址。
【计算机网络】【设备】路由器工作在网络层，转发依据是 IP 地址和路由表。
【计算机网络】【地址】MAC 地址标识局域网中的网卡。
【计算机网络】【地址】IP 地址标识 Internet 中的主机或接口。
【计算机网络】【地址】端口号标识主机上的应用进程。
【计算机网络】【地址】Socket 通常由 IP 地址和端口号构成，用于标识通信端点。
【计算机网络】【封装】数据从应用层向下传输时，每一层都可能增加首部或尾部。
【计算机网络】【封装】应用层数据越小，固定协议开销占比越大，传输效率越低。
【计算机网络】【时延】发送时延也叫传输时延，表示把全部数据推到链路上的时间。
【计算机网络】【时延】发送时延和数据大小、链路速率有关。
【计算机网络】【时延】传播时延表示信号在链路上传播的时间。
【计算机网络】【时延】传播时延和距离、信号传播速度有关，和帧大小无关。
【计算机网络】【文件传输】若题目给出握手延迟，总时间等于握手时延加文件发送时延加最后一位传播时延。
【计算机网络】【文件传输】若题目说握手耗费 2 个 RTT，则握手时延为 $2\times RTT$。
【计算机网络】【停止等待】停止等待协议中发送方每发送一帧，就必须等待 ACK 后才能发送下一帧。
【计算机网络】【停止等待】忽略 ACK 发送时延时，利用率 $U=\frac{\text{发送时延}}{\text{发送时延}+RTT}$。
【计算机网络】【停止等待】若题目给单向传播时延，则 $U=\frac{\text{发送时延}}{\text{发送时延}+2\times\text{传播时延}}$。
【计算机网络】【停止等待】提高停止等待利用率的方法包括使用滑动窗口或连续 ARQ、增大帧长、减少确认等待开销。
【计算机网络】【滑动窗口】滑动窗口充分利用链路时，窗口大小至少满足 $W\ge\frac{RTT+T_t}{T_t}$。
【计算机网络】【数据链路层】数据链路层功能包括封装成帧、帧同步、差错控制、流量控制、透明传输、链路管理、介质访问控制 MAC。
【计算机网络】【数据链路层】路由选择不属于数据链路层功能。
【计算机网络】【数据链路层】端到端可靠传输不属于数据链路层功能。
【计算机网络】【数据链路层】拥塞控制不属于数据链路层功能。
【计算机网络】【ACK】接收方正确收到数据帧后返回 ACK。
【计算机网络】【ACK】发送方超时未收到 ACK 时可能重传。
【计算机网络】【GBN】回退 N 帧 GBN 的发送窗口可以大于 1。
【计算机网络】【GBN】回退 N 帧 GBN 的接收窗口固定为 1。
【计算机网络】【GBN】回退 N 帧 GBN 出错后从出错帧开始全部重传。
【计算机网络】【GBN】GBN 序号字段为 $n$ 位时，最大发送窗口为 $2^n-1$。
【计算机网络】【SR】选择重传 SR 只重传出错或丢失的帧。
【计算机网络】【SR】选择重传 SR 接收方可以缓存乱序到达的帧。
【计算机网络】【SR】选择重传 SR 接收窗口可以大于 1。
【计算机网络】【SR】SR 序号字段为 $n$ 位时，最大窗口为 $2^{n-1}$。
【计算机网络】【SR】SR 窗口不能超过序号空间一半，是为了避免新帧和旧帧重传发生序号混淆。
【计算机网络】【CSMA/CD】CSMA/CD 用于传统有线以太网，核心是冲突检测。
【计算机网络】【CSMA/CD】CSMA/CD 为保证发送方能检测到冲突，必须满足 $2\times\text{传播时延}\le\text{最小帧发送时延}$。
【计算机网络】【CSMA/CD】以太网最小帧长为 $64\text{ Byte}=512\text{ bit}$。
【计算机网络】【CSMA/CD】10 Mbps 以太网发送 512 bit 的最小帧发送时延为 $51.2\text{ us}$。
【计算机网络】【CSMA/CD】二进制指数退避第 12 次碰撞时 $k=\min(12,10)=10$，范围为 $0\sim1023$。
【计算机网络】【CSMA/CA】CSMA/CA 用于无线局域网，核心是冲突避免。
【计算机网络】【CSMA/CA】无线局域网难以边发送边检测冲突，且存在隐藏站问题，因此不适合 CSMA/CD。
【计算机网络】【CSMA/CA】RTS/CTS 用于缓解隐藏终端问题。
【计算机网络】【CSMA/CA】RTS 是 Request To Send。
【计算机网络】【CSMA/CA】CTS 是 Clear To Send。
【计算机网络】【交换机】交换机根据目的 MAC 地址转发。
【计算机网络】【交换机】交换机学习 MAC 地址表时依据源 MAC 地址。
【计算机网络】【交换机】未知目的 MAC 时，交换机会泛洪。
【计算机网络】【交换机】每个交换机端口是一个独立冲突域。
【计算机网络】【交换机】未划分 VLAN 时，整个交换机通常是一个广播域。
【计算机网络】【VLAN】VLAN 是逻辑局域网，可以隔离广播域，与物理位置无关。
【计算机网络】【VLAN】IEEE 802.1Q 在以太网帧中插入 4 字节 VLAN Tag。
【计算机网络】【VLAN】VLAN ID 用于标识帧所属 VLAN。
【计算机网络】【ARP】ARP 用于根据 IP 地址查询 MAC 地址。
【计算机网络】【ARP】ARP 请求通常是广播。
【计算机网络】【ARP】ARP 响应通常是单播。
【计算机网络】【ARP】ARP 不属于 TCP 或 UDP。
【计算机网络】【STP】STP 生成树协议用于防止二层交换网络出现环路。
【计算机网络】【STP】STP 核心过程包括选举根桥、选择根端口、选择指定端口、逻辑阻塞部分冗余链路、形成无环树状拓扑。
【计算机网络】【CIDR】CIDR 使用 IP/前缀长度表示网络。
【计算机网络】【CIDR】主机位数等于 $32-\text{前缀长度}$。
【计算机网络】【CIDR】可用主机数等于 $2^{\text{主机位数}}-2$。
【计算机网络】【CIDR】网络地址等于 IP 地址 AND 子网掩码。

【计算机网络】【地址范围】网络地址是主机位全 0。
【计算机网络】【地址范围】广播地址是主机位全 1。
【计算机网络】【地址范围】可用地址范围是网络地址 + 1 到广播地址 - 1。
【计算机网络】【子网】判断两个 IP 是否同一子网，要分别计算网络地址。
【计算机网络】【子网】网络地址相同表示同一子网，可直接通信。
【计算机网络】【子网】网络地址不同表示不同子网，需要路由器转发。
【计算机网络】【子网划分】可划分子网数等于 $2^{\text{借用子网位数}}$，需大于等于需求子网数。
【计算机网络】【子网划分】可用主机数等于 $2^{\text{剩余主机位数}}-2$。
【计算机网络】【子网划分】新前缀等于原前缀加借用子网位数。
【计算机网络】【CIDR 聚合】CIDR 路由聚合的核心是找多个地址块的最长共同前缀。
【计算机网络】【CIDR 聚合】192.168.0.0/24 和 192.168.1.0/24 可聚合为 192.168.0.0/23。
【计算机网络】【路由表】路由器转发分组时采用最长前缀匹配。
【计算机网络】【路由表】若多条路由都能匹配目的地址，选择前缀最长的一条。
【计算机网络】【路由表】如果没有任何路由匹配，则使用默认路由。
【计算机网络】【路由器】一个 IP 数据报经过 $n$ 个路由器时，沿途设备数为 $n+2$。
【计算机网络】【路由器】一个 IP 数据报经过 $n$ 个路由器时，路由表查询次数为 $n$。
【计算机网络】【路由器】目的主机接收数据报后不再转发，因此不算转发表查询。
【计算机网络】【交换方式】电路交换建立连接、按序到达，典型特点是独占线路。
【计算机网络】【交换方式】报文交换不建立连接，不一定按序到达，以完整报文为单位存储转发。
【计算机网络】【交换方式】虚电路分组交换建立连接，通常按序到达，所有分组沿同一路径。
【计算机网络】【交换方式】数据报分组交换不建立连接，不保证按序到达，每个分组独立选择路径。
【计算机网络】【MTU】MTU 是最大传输单元。
【计算机网络】【IP 分片】IP 分片题中 MTU 通常表示链路允许的 IP 数据报最大总长度。
【计算机网络】【IP 分片】每个 IP 分片都有 IP 首部。
【计算机网络】【IP 分片】每个 IP 分片总长度不能超过 MTU。
【计算机网络】【IP 分片】除最后一片外，数据部分长度必须是 8 字节的整数倍。
【计算机网络】【IP 分片】片偏移单位是 8 字节。
【计算机网络】【IP 分片】分片可由源主机或中间路由器完成。
【计算机网络】【IP 分片】IP 重组只在目的主机完成。
【计算机网络】【IP 分片】原始数据部分等于 IP 数据报总长度减 IP 首部长度。
【计算机网络】【IP 分片】每片最大数据等于 MTU 减 IP 首部长度。
【计算机网络】【IP 分片】非最后一片数据长度等于每片最大数据向下取 8 的倍数。
【计算机网络】【IP 分片】片偏移等于本片数据起始字节除以 8。

【计算机网络】【NAT】NAT 可缓解 IPv4 地址不足。
【计算机网络】【NAT】NAT 可隐藏内部网络结构。

【计算机网络】【AS】AS 内部使用 IGP，例如 RIP、OSPF。
【计算机网络】【AS】AS 之间通常使用 BGP。
【计算机网络】【AS】每个 AS 有唯一 AS 编号 ASN。
【计算机网络】【路由协议】RIP 是 AS 内部 IGP，使用距离向量，常基于 Bellman-Ford 思想。
【计算机网络】【路由协议】OSPF 是 AS 内部 IGP，使用链路状态，Flooding 传播 LSA，使用 Dijkstra 算法。
【计算机网络】【路由协议】BGP 是 AS 之间 EGP，使用路径向量，使用 TCP 179 端口。
【计算机网络】【OSPF】OSPF 中 LSA 是链路状态通告。
【计算机网络】【OSPF】OSPF 中 LSDB 是链路状态数据库。
【计算机网络】【OSPF】OSPF 中 Flooding 表示泛洪。
【计算机网络】【OSPF】OSPF 使用 Dijkstra 最短路径算法。
【计算机网络】【距离向量】距离向量路由算法公式：$D_C(X)=\min\{\text{C 到邻居的代价}+\text{邻居到 X 的代价}\}$。
【计算机网络】【传输层】发送端复用是多个应用进程的数据通过传输层交给网络层。
【计算机网络】【传输层】接收端分用是传输层根据端口号把数据交给正确应用进程。
【计算机网络】【UDP】UDP 无连接。

【计算机网络】【UDP】UDP 开销小、速度快。
【计算机网络】【UDP】UDP 常用于 DNS、语音、视频、实时通信等。
【计算机网络】【UDP】UDP 端口号用于区分同一主机上的不同应用进程。
【计算机网络】【TCP】TCP 面向连接。
【计算机网络】【TCP】TCP 提供可靠传输。
【计算机网络】【TCP】TCP 面向字节流。
【计算机网络】【TCP】TCP 支持全双工通信。

【计算机网络】【TCP】TCP 不是面向消息的。
【计算机网络】【TCP】TCP 不保留应用层消息边界。
【计算机网络】【TCP/UDP】TCP 面向连接、可靠、面向字节流；UDP 无连接、不保证可靠、开销小、速度快。
【计算机网络】【TCP 序号】SEQ 是本报文段数据部分第一个字节的编号。
【计算机网络】【TCP 确认】ACK 是接收方期望收到的下一个字节编号。
【计算机网络】【TCP 确认】正常收到时 $ACK=\text{对方 SEQ}+\text{有效载荷长度}$。
【计算机网络】【TCP 确认】若前面数据丢失，即使后面数据到达，确认号仍然是第一个缺失字节的序号。
【计算机网络】【TCP 确认】ACK 不是已经收到的最后一个字节，而是期望收到的下一个字节。
【计算机网络】【TCP 可靠】TCP 可靠传输依靠序号机制、确认机制 ACK、校验和、超时重传、滑动窗口、流量控制、拥塞控制。
【计算机网络】【TCP 可靠】TCP 通过序号和确认机制保证数据按序到达。
【计算机网络】【TCP 可靠】TCP 通过校验和检测传输错误。
【计算机网络】【TCP 可靠】TCP 通过超时重传处理丢失报文段。
【计算机网络】【TCP 可靠】TCP 通过滑动窗口和接收窗口实现流量控制。
【计算机网络】【TCP 可靠】TCP 通过慢启动、拥塞避免、快重传、快恢复实现拥塞控制。
【计算机网络】【TCP 流量控制】流量控制目的是防止发送方发送过快，导致接收方缓冲区溢出。
【计算机网络】【TCP 流量控制】TCP 流量控制通过滑动窗口和接收窗口 rwnd 实现。
【计算机网络】【TCP 流量控制】流量控制保护接收方。
【计算机网络】【TCP 拥塞控制】拥塞控制目的是防止过多数据进入网络造成网络拥塞。
【计算机网络】【TCP 拥塞控制】拥塞控制保护网络。
【计算机网络】【TCP 拥塞控制】拥塞控制四个核心机制是慢开始、拥塞避免、快重传、快恢复。
【计算机网络】【TCP 拥塞控制】慢开始阶段窗口指数增长，例如 1、2、4、8、16。
【计算机网络】【TCP 拥塞控制】拥塞避免阶段窗口线性增长，例如 8、9、10、11。
【计算机网络】【TCP 拥塞控制】发生超时时，$ssthresh=\frac{\text{当前 cwnd}}2$，$cwnd=1\text{ MSS}$，重新慢开始。
【计算机网络】【TCP 拥塞控制】收到 3 个重复 ACK 时触发快重传。
【计算机网络】【TCP 拥塞控制】收到 3 个重复 ACK 时，$ssthresh=\frac{\text{当前 cwnd}}2$，通常进入快恢复或拥塞避免。
【计算机网络】【TCP 拥塞控制】慢启动前 $n$ 轮累计最多发送 $2^n-1$ 个报文段。
【计算机网络】【TCP 对比】流量控制保护接收方，拥塞控制保护网络。
【计算机网络】【序号空间】为避免序号回绕导致新旧报文混淆，在 MSL 时间内不能重复使用序号。
